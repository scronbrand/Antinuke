# Contributing Guidelines

Thank you for considering contributing to **Antinuke**! We welcome bug reports, feature suggestions, and pull requests.

## How to Open an Issue

1. **Search existing issues** – make sure the problem hasn't already been reported.
2. **Provide a clear title** – summarize the problem or feature in a few words.
3. **Describe the problem**
   - What you expected to happen.
   - What actually happened.
   - Steps to reproduce (if applicable).
   - Include any relevant screenshots or logs.
4. **Environment details** – Discord.js version, Node.js version, OS, and any configuration differences.
5. **Label the issue** – use appropriate labels (`bug`, `enhancement`, `question`).

## Pull Requests

- Fork the repository and create a new branch for your changes.
- Follow the existing code style (TypeScript, ESLint, Prettier).
- Ensure the project builds and all tests pass (`npm test`).
- Include a brief description of what the PR does and reference any related issue.

## Code Style

- Use **TypeScript** with strict typing.
- Follow the project's existing naming conventions.
- Run `npm run lint` before submitting.

## Support

If you need help, feel free to join our Discord server or open a discussion thread.

# Руководство по внесению вклада

Спасибо, что хотите помочь развитию **Antinuke**! Мы приветствуем сообщения об ошибках, предложения новых функций и pull‑request'ы.

## Как открыть Issue

1. **Поиск существующих Issue** – убедитесь, что проблема ещё не задокументирована.
2. **Чёткое заглавие** – в нескольких словах опишите проблему или запрос.
3. **Описание проблемы**
   - Что вы ожидали увидеть.
   - Что произошло на самом деле.
   - Шаги для воспроизведения (если применимо).
   - Прикрепите скриншоты или логи, если они помогают.
4. **Информация о среде** – версия Discord.js, Node.js, ОС и любые отличия в конфигурации.
5. **Теги Issue** – используйте подходящие метки (`bug`, `enhancement`, `question`).

## Pull Request'ы

- Сделайте fork репозитория и создайте новую ветку для изменений.
- Соблюдайте стиль кода проекта (TypeScript, ESLint, Prettier).
- Убедитесь, что проект собирается и все тесты проходят (`npm test`).
- Добавьте краткое описание изменений и при необходимости ссылку на Issue.

## Стиль кода

- Пишите код на **TypeScript** со строгой типизацией.
- Следуйте существующим соглашениям об именовании.
- Перед отправкой запустите `npm run lint`.

## Поддержка

Если нужна помощь, присоединяйтесь к нашему Discord‑серверу или откройте обсуждение в репозитории.

Thank you for considering contributing to **Antinuke**! We welcome bug reports, feature suggestions, and pull requests.
